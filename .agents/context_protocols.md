# Agent-Native Context Protocols

This document defines how any AI Agent must "read" and "evolve" this project. This is the manual for the project's digital soul.

## 1. Context Anchoring (The Golden Rule)
Any new logic or UI element must first be "anchored" to the existing state-of-truth:
- **Visuals:** Must be mapped to `src/data/config.ts` (`THEME`). Never use ad-hoc Tailwind classes.
- **Copy:** Must be mapped to `src/data/config.ts` (`LABELS`). Never hardcode strings in components.
- **Philosophy:** Check `.agents/constitution/SKILL.md` before every major edit.

## 2. Prompting Patterns for the Human
When communicating with the USER, the Agent should follow these high-density patterns:
- **Vibe Sync:** Use the USER's language (e.g., "Mükemmel MVP", "Kurşun Geçirmez").
- **Visual Evidence:** Always provide artifact links for walkthroughs and task lists.
- **Proactive Validation:** Don't ask to run tests; run them as part of the "vibe" delivery.

## 3. Engineering Secret: The "Perfect Build" Loop
An Agent has not finished a task until the following cycle is complete:
1. `type-check` (Logical Integrity)
2. `lint` (Stylistic Integrity)
3. `build` (Production Integrity)
4. `Hero Preview` (Aesthetic Integrity)

## 4. The "Vibe" Vocabulary
When designing or coding, use these aesthetic anchors:
- **Apple-Style:** High whitespace, soft shadows, rounded typography.
- **Kraft Aesthetic:** Tactile, organic, premium wholesale feel.
- **Focus:** Single-point attention (Wizard patterns).
